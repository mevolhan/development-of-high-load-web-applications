# Лабораторные работы по высоконагруженным системам

## Лабораторная работа №1: Балансировка и отказоустойчивость с Nginx

**Описание:**  
Разработано Java-приложение на Spring Boot с эндпоинтом `/api/counter`, возвращающим увеличивающийся счётчик. Три экземпляра приложения запущены в Docker, Nginx в роли балансировщика нагрузки.

**Технологии:**  
Java 17, Spring Boot, Docker, Nginx, JMeter

### Алгоритмы балансировки
- Round Robin (по умолчанию)
- Least Connections (`least_conn`)
- IP Hash (`ip_hash`)
- Random (`random`)

### Отказоустойчивость
Настроены параметры `max_fails=2` и `fail_timeout=30s` для автоматического исключения отказавшего сервера и его восстановления.

### Тестирование
Нагрузочное тестирование проведено с помощью Apache JMeter.

## Лабораторная работа №2: Разделяемое хранилище данных с Redis

## Описание
Разработано Java-приложение на Spring Boot с двумя функциональными модулями:
- **Счётчик запросов** — общее значение хранится в Redis и увеличивается при каждом обращении к `/api/counter`.
- **Таблица лидеров (Leaderboard)** — на основе Sorted Set Redis реализован REST API для добавления очков игроков и получения топа.

Три экземпляра приложения запущены в Docker, Nginx выступает в роли балансировщика нагрузки, Redis — единое хранилище состояния.

## Технологии
- Java 17, Spring Boot, Spring Data Redis
- Docker, Docker Compose
- Nginx
- Redis

## Реализация
- **Redis** используется как разделяемое хранилище для счётчика и лидерборда.
- **Sorted Set** обеспечивает автоматическую сортировку игроков по очкам.
- **REST API**:
  - `POST /leaderboard/{playerId}?score=` — добавить/обновить очки.
  - `GET /leaderboard/top?limit=` — получить топ N игроков.
  - `GET /leaderboard/rank/{playerId}` — узнать ранг игрока.
- **Docker Compose** поднимает три экземпляра приложения, Redis и Nginx в единой сети.

## Проверка работы
- Счётчик инкрементируется независимо от того, на какой экземпляр попал запрос.
- Данные лидерборда сохраняются в Redis и доступны всем экземплярам.
- Балансировка Nginx распределяет запросы между тремя контейнерами.
